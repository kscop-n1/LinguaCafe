# LinguaCafe Public Layer Overview (`public/`)

This directory is the web root containing compiled assets, static media, fonts, and the HTTP request entry point.

---

## 1. Directory Structure

* [index.php](file:///c:/q/git/linguacafe/LinguaCafe/public/index.php): Core entry point file for the web server. Bootstraps Laravel and handles incoming HTTP requests.
* [manifest.json](file:///c:/q/git/linguacafe/LinguaCafe/public/manifest.json): Configuration file listing icons, names, and parameters to enable Progressive Web App (PWA) installation.
* [.htaccess](file:///c:/q/git/linguacafe/LinguaCafe/public/.htaccess): Configuration for Apache servers containing rewrite directives to funnel pretty URLs through `index.php`.
* **`css/`** & **`js/`**: Contain compiled, minified, versioned CSS and JS files generated by Laravel Mix compilation (`webpack.mix.js`).
* **`images/`**: Houses application images:
  * **`flags/`**: Contains country flags mapped to languages in the target translation selector.
  * Default placeholder graphics for book covers, reviews, etc.
* **`fonts/`** & **`webfonts/`**: Store loaded web fonts such as `@mdi/font` icons.
* **`default/`**: Default resource templates.

---

## 2. Compilation Workflow

Files inside **`css/`** and **`js/`** should not be edited directly. To rebuild these files after making modifications to frontend components, run the following scripts:
* **Development Build**:
  ```bash
  npm run dev
  ```
* **Production Build**:
  ```bash
  npm run prod
  ```
* **Watch Mode**:
  ```bash
  npm run watch
  ```
