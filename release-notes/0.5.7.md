# LinguaCafe 0.5.7

This release hardens container startup so the webserver waits for the database before running migrations and seeding.

## Fixed

- Added an explicit database socket wait step to the webserver entrypoint before `php artisan migrate --force` runs.
- Added the same database wait step to the development entrypoint so dev and production startup follow the same readiness path.
- Kept the existing migration retry loop in place after the readiness wait, so transient startup delays are handled more cleanly.

## Validation

- Ran the updated entrypoint inside the live `linguacafe-webserver` container and confirmed it waits for `host.docker.internal:3308` before migrating.
- Verified the dry-run reaches `Nothing to migrate` and database seeding without the earlier connection-refused noise.
