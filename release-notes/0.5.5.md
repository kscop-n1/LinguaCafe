# LinguaCafe 0.5.5

This release continues the migration stabilization work with a focus on theme/state consistency in the deployed app.

## Fixed

- Synced theme selection back to the theme cookie so the server bootstrap and client shell now agree on the active theme.
- Fixed the theme settings editor so `eink` is handled explicitly instead of collapsing into the light editor path.
- Fixed the theme settings loader so saved `eink` colors are loaded and saved alongside `light` and `dark`.
- Removed the duplicate root route that returned only the Laravel version before the authenticated home shell route.
- Unified the authenticated home page theme fallback with the login page fallback when no theme cookie is present.

## Validation

- Verified the live browser can log in, switch to `eink`, reload, and stay on `v-theme--eink`.
- Verified the server-rendered `theme-color` matches the selected theme after reload.
- Verified the default no-cookie bootstrap now renders the same light theme on login and home.
