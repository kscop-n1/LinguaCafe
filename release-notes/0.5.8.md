# LinguaCafe 0.5.8

This release removes the unintended password-change gate for existing migrated users by backfilling the live user state.

## Fixed

- Backfilled existing users so `password_changed` is `true` for migrated/live accounts that were already in the database.
- Left the existing admin-created-user flow intact so newly created admin users can still be prompted to change their password.

## Validation

- Ran the backfill migration in the live `linguacafe-webserver` container.
- Verified the home shell no longer shows the password-change gate or button for the current live account.
- Verified the browser bootstrap remains clean after the migration.
