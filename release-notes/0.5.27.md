# LinguaCafe 0.5.27

## Changed
- The legacy auth views now share the same cookie-based theme-color shell as the migrated app instead of relying on the old plain Bootstrap layout.
- The dormant register/reset/confirm/verify views now render safely even when their old route names are not enabled in this build.
- The fallback links in those views now use the relative `/login` path so they stay correct in the live deployment.

## Verified
- Rendering `auth.register`, `auth.passwords.email`, `auth.passwords.reset`, `auth.passwords.confirm`, and `auth.verify` inside the live `linguacafe-webserver` container no longer throws missing-route errors.
- The rendered views include the same cookie-based `theme-color` handling as the migrated shell.
- The fallback links in the render output point at `/login` instead of `localhost/login`.
