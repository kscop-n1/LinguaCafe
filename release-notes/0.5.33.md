# LinguaCafe 0.5.33

## Fixed
- Fixed the root theme mismatch where an old theme cookie could force Vuetify into dark mode while local settings still showed light mode.
- Fixed Vuetify hover/focus overlays on buttons, list items and fields so controls no longer become solid black when hovered or focused.
- Migrated remaining Vuetify 2-era `theme="dark"`, `:dark` and button `text` props to Vuetify 3-supported theme/variant behavior. This keeps dialogs, alerts, toolbar controls and sidebar buttons on the current app theme.
- Removed remaining Popper/Bootstrap Vite leftovers from the frontend build configuration.

## Verified
- npm run production
- ./vendor/bin/phpunit --filter VueMigrationStaticTest
- Browser verification on https://lingua.lan for theme cookie/localStorage sync, dropdown active state, filter button hover overlay, dialog selected state and sidebar icons/buttons.
