# LinguaCafe 0.5.6

This release tightens theme bootstrap behavior and keeps the deployed runtime aligned with the actual live topology.

## Fixed

- Re-enabled live OS theme watching in the app shell so `auto` responds when the system color scheme changes.
- Restored the active Vuetify theme bootstrap order so saved `light`, `dark`, and `eink` themes initialize correctly on reload.
- Kept the theme cookie and local storage in sync during auto/theme selection changes.
- Aligned the checked-in Docker Compose defaults with the live deployment topology that uses the external MySQL host.

## Validation

- Verified the live browser booted cleanly on `https://lingua.lan` after redeploying the rebuilt assets.
- Verified `eink` still boots as `v-theme--eink` with `theme-color` set to `#FFFFFF` after reload.
- Verified `auto` resolves to `light` in this browser session without bootstrap errors and synchronizes the cookie on the follow-up reload.
- Verified `/review`, `/books`, and `/user-manual` load cleanly with the deployed assets.
