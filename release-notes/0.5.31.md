# LinguaCafe 0.5.31

## Fixed
- Fixed remaining active Vuetify control states so selected tabs, buttons and fields no longer render as black blocks after the migration.
- Fixed mobile bottom navigation, login inputs, user manual tree, dropdowns and overlay surfaces so foreground/background colors remain readable in light and dark themes.
- Fixed Varela font bundling by moving it to a Vite-managed source asset and using the fingerprinted build output.

## Changed
- Removed legacy Bootstrap, jQuery and Popper frontend dependencies left over from the old Vue 2 stack.
- Migrated Sass imports to supported module syntax and removed Bootstrap Sass usage.
- Expanded static migration checks for old Vuetify overlay props, theme variables, activator binding, legacy text utilities and Vite font asset handling.

## Verified
- npm run production
- ./vendor/bin/phpunit --filter VueMigrationStaticTest
- Browser verification on https://lingua.lan for active tabs/buttons, select/dropdown overlays, mobile bottom navigation, login inputs, user manual tree and overlay dialogs.

## Known test environment note
- Full PHPUnit in the ad-hoc container still requires the linguacafe-dev database hostname/network; the focused migration test passes.
