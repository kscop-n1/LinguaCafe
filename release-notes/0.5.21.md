# LinguaCafe 0.5.21

## Changed
- Centralized the theme bootstrap around the cookie-first server/client contract so login and home now default to light consistently and the client no longer starts from a stale localStorage-only value.
- Kept auto theme live-following the system color scheme by resolving the active theme from `prefers-color-scheme` when auto mode is enabled.
- Passed login-page theme settings through the Blade layout prop when available instead of hardcoding an empty object.

## Verified
- Fresh login without a theme cookie loads with a light shell and `meta[name=theme-color]` set to `#F2F3F5`.
- Fresh login with `theme=eink` loads with `v-theme--eink` and `meta[name=theme-color]` set to `#FFFFFF`.
- Auto mode with a dark system preference reloads to `v-theme--dark` and `meta[name=theme-color]` set to `#28272C`.
- Authenticated home after login loads cleanly with `window.__LINGUACAFE_BOOTSTRAP_ERROR === null`.
