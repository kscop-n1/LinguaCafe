# LinguaCafe 0.5.2

This release focuses on stabilizing the Vue 3 / Vuetify 3 migration and making the production Docker images buildable for GitHub Container Registry.

## Fixed

- Fixed Vuetify activator slot usage across dialogs, menus, tooltips, and popovers so production builds no longer crash on missing `aria-expanded` activator attributes.
- Replaced legacy `$vuetify.theme.currentTheme` access with a compatibility helper for migrated Vue components.
- Guarded DOM event binding in mounted hooks so missing optional elements no longer throw `addEventListener` errors.
- Fixed library book opening when a requested book id is not present in the loaded list.
- Fixed vocabulary chapter filtering when no valid book is selected.
- Fixed statistics loading when the default reading goal record is missing.
- Removed stray debug logging from theme selection.

## Docker / Deployment

- Added `.dockerignore` so local MySQL data, dependency folders, build output, and runtime cache are not sent to Docker build context.
- Fixed the PHP webserver image build by running Composer package discovery with a build-time `BROADCAST_DRIVER=log` override. Runtime broadcasting configuration remains environment-driven.
- Added Docker-safe Laravel defaults for image-only installs: MySQL as the default database, file-backed sessions/cache, Predis, the compose Redis host, a fallback app key, and fallback Pusher/Reverb values. Environment variables still override these defaults.
- Trust forwarded proxy headers so HTTPS reverse-proxy installs generate HTTPS Vite asset URLs instead of browser-blocked mixed-content HTTP assets.
- Verified local production image builds for:
  - `ghcr.io/kscop-n1/linguacafe-webserver:v0.5.2`
  - `ghcr.io/kscop-n1/linguacafe-webserver:0.5.2`
  - `ghcr.io/kscop-n1/linguacafe-webserver:latest`
  - `ghcr.io/kscop-n1/linguacafe-python-service:v0.5.2`
  - `ghcr.io/kscop-n1/linguacafe-python-service:0.5.2`
  - `ghcr.io/kscop-n1/linguacafe-python-service:latest`

## Validation

- `npm run production` passes.
- Authenticated browser smoke check passed locally for Dashboard, Books, and Vocabulary Search with an empty browser console after page load and interaction.
- Docker webserver production build passed inside `docker/PhpDockerfile`.
- Docker python-service image build passed using `docker/PythonDockerfile`.
- Recreated the local compose `webserver` container from `ghcr.io/kscop-n1/linguacafe-webserver:latest` and verified `/` returns `302` to `/login`, `/login` returns `200`, the built JS asset returns `200`, and Horizon stays in `RUNNING` state.

## Known Notes

- The frontend build still emits Sass deprecation warnings from the older Bootstrap/SCSS stack.
- The build still leaves `/fonts/VarelaRound-Regular.ttf` as a runtime-resolved asset reference.
- GitHub Actions release workflow publishes GHCR tags from the created release tag. Existing repo tag style is `v0.5.2`; the local build also includes plain `0.5.2` for convenience.
