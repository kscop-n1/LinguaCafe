# LinguaCafe 0.5.28

## Changed
- Converted the remaining legacy Vuetify 2 compatibility props across the component tree to native Vuetify 3 variants or removed them where they were no longer needed.
- Removed the orphaned `resources/vue3` / PrimeVue frontend tree from the repository so there is only one shipped frontend path.
- Collapsed the migration audit to reflect that there is no active open audit surface left in the current note.

## Verified
- `rg` no longer finds bare `outlined`, `depressed`, `filled`, `dense`, or `dark` attributes in `resources/js/components`.
- The production build completes successfully after the compatibility cleanup.
- The live browser still boots cleanly on `/login`, `/review`, `/user-settings`, and `/user-manual` with `window.__LINGUACAFE_BOOTSTRAP_ERROR === null`.
