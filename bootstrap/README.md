# LinguaCafe Bootstrap Layer Overview (`bootstrap/`)

This directory is responsible for initializing and booting the Laravel application framework.

---

## 1. Directory Structure

* [app.php](file:///c:/q/git/linguacafe/LinguaCafe/bootstrap/app.php): The application configuration entry point. Using the Laravel 11 application builder, it configures:
  * **Routing**: Maps standard web endpoints (`routes/web.php`), CLI console commands (`routes/console.php`), and real-time broadcasting channels (`routes/channels.php`).
  * **Middleware**: Configures stateful API authentication for frontend requests via Sanctum, registers email verification logic, and registers custom middleware aliases like `admin` targeting [AdminMiddleware.php](file:///c:/q/git/linguacafe/LinguaCafe/app/Http/Middleware/AdminMiddleware.php).
  * **Exceptions**: Custom handler bindings for application exceptions (currently default).
* [providers.php](file:///c:/q/git/linguacafe/LinguaCafe/bootstrap/providers.php): Specifies active global service providers, registering `AppServiceProvider` and the Laravel Horizon queue dashboard helper `HorizonServiceProvider`.
* **`cache/`**: Contains runtime cache files generated by Laravel (such as compiled route structures or config maps) to optimize production response latency. This directory is empty in the repository except for a `.gitignore` template.

---

## 2. Developer Notes

* **Custom Middleware Registration**: If new middleware files are created under `app/Http/Middleware/`, they must be aliased or appended to global middleware stacks inside [app.php](file:///c:/q/git/linguacafe/LinguaCafe/bootstrap/app.php) to take effect.
* **Service Providers**: Any new service provider must be registered inside [providers.php](file:///c:/q/git/linguacafe/LinguaCafe/bootstrap/providers.php) to be loaded during application boot.
